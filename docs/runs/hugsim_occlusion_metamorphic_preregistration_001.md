# HUGSIM CF-O 001 可控遮挡形变实验预注册

> 状态：`preregistered_not_run`。本文件、配套 JSON、六份场景配置、分析器和
> 单元测试必须先提交，再启动仿真；运行后不回改判据。

## 目的与边界

验证 CF-O 第一部分：在目标车辆、ego、相机、背景和时间保持配对时，HUGSIM
能否独立改变一个更近车辆的横向位置，并使目标车辆的直接可见 RGB 支撑随几何
遮挡增强而不增加。

本实验检查的是 `scene-0383`、`CAM_FRONT`、同一种车辆资产和两个离散遮挡位置下
的内部几何与渲染机制。它不验证真实相机成像、遮挡比例的现实准确度、检测器、
目标身份保持、AD 响应或 HUGSIM 整体可信性。

## 六条件因子设计

目标车固定在 ego 前方 24 m、横向 0 m；遮挡车固定在前方 20 m，`partial` 横向
右移 2.0 m，`strong` 横向 0 m。两车均为 yaw 0、恒速 1 m/s，因此初始中心纵向
间隔 4 m 并在运行中保持。两车长度约 3.58 m，声明状态下不相交。

| 条件 | actor | 用途 |
|---|---|---|
| `no_actor` | 无 | 场景基线 |
| `target_only` | 目标车 | 无遮挡目标贡献 |
| `partial_occluder_only` | 偏置遮挡车 | partial 的遮挡车自身贡献 |
| `partial_both` | 偏置遮挡车＋目标车 | partial 组合画面 |
| `strong_occluder_only` | 居中遮挡车 | strong 的遮挡车自身贡献 |
| `strong_both` | 居中遮挡车＋目标车 | strong 组合画面 |

所有条件固定场景、资产、速度、ego 初始条件、确定性 plan writer 和 36 个仿真
step。`both` 条件中 actor 0 是遮挡车、actor 1 是目标车；单 actor 条件中 actor 0
是该条件唯一角色。运行间必须核对相同时间戳、ego 状态、动作和计划。

采用六条件而不是只比较三张组合图，是为了用配对差分分离角色贡献：

```text
无任何遮挡的目标变化 = target_only - no_actor
partial 下的目标变化 = partial_both - partial_occluder_only
strong 下的目标变化 = strong_both - strong_occluder_only
```

这样不会把“遮挡车自身产生了很多像素”误写成“目标更可见”。

## 预声明判据

### 1. 干预与几何入口条件

- 对应条件间，目标车或遮挡车的保存 box 状态最大绝对差不超过 `1e-8`；
- 六次运行的 ego、动作、计划和时间戳完全配对；
- 保存 box 中的目标—遮挡车平面偏移与声明的纵向 4 m、横向 2/0 m 一致，残差
  不超过 `1e-8 m`；
- 两个 `both` 条件中目标车与遮挡车的平面 footprint 净距始终大于 0.1 m，避免
  用相互穿插的 actor 制造遮挡；
- 在 `CAM_FRONT` 相机坐标中，遮挡车中心深度始终小于目标车中心深度，且两者
  中心深度均大于 0.2 m；
- 用独立投影代码把两个 3D box 的八角点投到 `CAM_FRONT`，以凸包交集除以目标
  凸包面积得到几何覆盖率。允许地面高度和栅格化造成少量非嵌套，但 partial 独有
  遮挡像素不得超过目标投影面积的 2%；这些像素从后续 RGB 支撑比较域中排除，
  使实际受评区域内的遮挡关系严格嵌套。每个有效帧还必须满足：

```text
strong projected coverage > partial projected coverage > 0
```

任何有效帧违反以上关系，都会把相应的“干预按声明发生/遮挡几何顺序正确”窄
主张裁决为 `rejected`。这里的 box 投影是几何代理，不是真实可见表面的精确真值。

### 2. 目标直接可见 RGB 支撑

对每个配对差分，以 RGB 三通道最大绝对差 `> 10` 的像素定义“发生可观察变化”。
测量只限于目标 3D box 在该帧投影出的凸包，再排除 partial 有而 strong 没有遮挡
的少量非嵌套区域；以 `target_only - no_actor` 的变化像素作为该帧目标参考支撑。
若参考支撑少于 25 像素，该帧标为 `unavailable`，不得当作通过。

partial/strong 的直接目标支撑率为：

```text
|遮挡条件中的目标变化像素 ∩ 无遮挡目标参考变化像素|
----------------------------------------------------
            |无遮挡目标参考变化像素|
```

逐个有效帧预期 `partial support >= strong support`，允许相等；任何反转都拒绝
“遮挡增强不增加目标直接支撑”这一方向主张。为了证明干预不是完全无响应，还要求
整个窗口的 `median(partial) > median(strong)`；否则也裁决为 `rejected`。

`unoccluded support >= occluded support` 由交集定义自动成立，只作为量纲和覆盖
诊断，**不计为支持规律的证据**。检测置信度不要求单调，也不进入本实验。

另用 `CAM_FRONT` 顶部 20%（不与这些 actor 投影相交）作为配对稳定性控制带。
六条件相对 `no_actor` 或其因子控制的配对差分中，控制带内 `> 10` 的变化像素率
每帧不得超过 `1e-4`；超出时说明跨运行渲染漂移可能污染目标差分，该帧不得用于
接受可见性主张。这个门槛只审计配对稳定性，不是图像真实性指标。

### 3. 已知负例

单元测试固定两个核心反例：

1. 交换 partial 与 strong 的支撑序列，单调指标必须检出反转；
2. 把遮挡车中心放到目标车之后，深度顺序检查必须失败。

另检查投影覆盖率顺序和参考像素不足时的 `unavailable` 分支。负例只证明分析器
能识别预定义错误，不代表覆盖所有遮挡与渲染失效。

## 证据独立性与裁决

相机投影、box 覆盖和 RGB 差分不用 HUGSIM semantic、depth、NC/TTC/PDMS，具有
计算独立性；但相机标定、box、RGB 和两辆车仍全部由 HUGSIM 产生。因此它只能
检查 HUGSIM 自己声明状态之间的内部因果一致性，不能证明与真实传感器等效。

- 入口条件与几何顺序全部满足：接受该窄几何主张；否则 `rejected`；
- 全部有效帧无支撑反转、窗口中位数严格下降且无缺测：接受该窄可见性主张；
- 有缺测但没有已观测反转：只可 `down-weighted`；
- 出现反转或中位数不下降：可见性主张 `rejected`；
- 即使两个窄主张均接受，整体证据仍为 `down-weighted`，因为缺少来源独立真值。

允许的最强主张是：在本场景、相机、资产、轨迹和两个预声明遮挡位置内，保存
状态的几何遮挡顺序正确，并且渲染 RGB 中目标直接可见支撑沿预期方向变化。

不得升级为：遮挡的真实数值准确、真实相机一致、检测/跟踪正确、AD 对遮挡响应
正确、闭环安全、real–sim 等效或 HUGSIM 普遍可信。
